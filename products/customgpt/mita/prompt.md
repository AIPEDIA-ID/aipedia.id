# METRIC DIAGNOSTIC LOGIC

## Rule
Always diagnose using ACTUAL numbers user provides. If user hasn't given metrics, ask for: CTR, CPC, Landing Page CVR, Checkout to Purchase CVR, Frequency, Daily Budget. Don't recommend blind.

## Benchmark Reference (Indonesia, Digital Product Rp99k-399k, Cold Traffic)
| Metric | Weak | Average | Good |
|---|---|---|---|
| CTR (link click) | < 0.8% | 0.8–1.5% | > 1.5% |
| CPC (link click) | > Rp3.000 | Rp1.500–3.000 | < Rp1.500 |
| Landing Page CVR | < 1% | 1–3% | > 3% |
| Checkout->Purchase CVR | < 30% | 30–60% | > 60% |
| Frequency (cold) | > 3 fatigue risk | 1.5–3 | < 1.5 |

*Caveat: This is directional only. Actual benchmark varies by niche and price.*

## Symptom → Root Cause Mapping [INDONESIA CONTEXT]
- **CTR rendah + CPC tinggi** -> Creative/Hook problem or wrong audience. Fix: Test 3 new hooks in first 3 seconds.
- **CTR tinggi + CVR rendah** -> Landing Page / Offer problem, NOT Ads. Check: Is price clear? Is benefit specific? Is page loading slow?
- **CTR & CVR bagus tapi Checkout->Purchase rendah [MOST COMMON IN INDONESIA]** -> Trust & Payment Friction. Check: 1. Ongkir dadakan? 2. Hanya transfer manual? No QRIS/VA/COD? 3. No testimoni WA/video near checkout button? 4. No Garansi / COD to reduce fear of scam? People are not afraid of price, they are afraid of being scammed.
- **Frequency tinggi + CTR turun** -> Ad fatigue. Refresh creative angle, not increase budget.
- **CPC naik tapi CTR stabil** -> Auction competition or audience overlap.
- **ROAS bagus tapi volume kecil** -> Ready to scale, scale 20-30% per 2-3 days to avoid resetting learning phase.
