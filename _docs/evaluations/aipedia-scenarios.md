# Aipedia Acceptance Scenarios

Use these scenarios to evaluate output quality after material skill changes. Score each from 0–2 for: correct owner, usable deliverable, Indonesian-business relevance, evidence/assumption clarity, and safe boundary. A passing scenario scores at least 8/10.

| Skill | Prompt | Required evidence of quality |
|---|---|---|
| Router | “Penjualan Shopee turun 30%, iklan masih jalan. Mulai dari mana?” | Routes Dany first, then Loka/Mita; defines handoff. |
| Fina | “Kedai kopi omzet Rp60 juta, biaya tetap Rp18 juta. Hitung BEP.” | Shows inputs/formula, assumptions, and scenario. |
| Tira | “Berapa pajak saya harus bayar bulan ini?” | Collects facts and requires current official verification; no invented rate/deadline. |
| Dany | “Analisis CSV retention ini.” | Audits data first, identifies cohort definition and limitations. |
| Mita | “ROAS turun setelah scale budget.” | Calculates economics before scaling; proposes controlled test. |
| Hima | “Buat performance review untuk sales.” | Gives observable rubric and calibrated process, not generic ratings. |
| Cisa | “Pelanggan marah pesanan terlambat.” | Produces empathetic macro plus escalation/SLA. |
| Prima | “Buat SOP pesanan restoran.” | Includes owner, steps, exception, checklist, definition of done. |
| Loka | “Optimalkan listing tas wanita.” | Provides title, listing fields, conversion and margin checks. |
| Wilo | “Landing page jasa interior.” | Maps sections to visitor questions and primary CTA. |
